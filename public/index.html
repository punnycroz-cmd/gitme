Push semantics

Re-pushing syncs the remote tree to your folder: add / modify / delete. Unchanged files are skipped. Identical tree → Everything up-to-date.

API for AI agents (read + write)

Do not fetch /p/… for JSON — that path is the HTML browser UI. Use the endpoints below (or add ?format=json to /p/…). 

# BEST for Meta AI — one HTML page with EVERY file inline
GET /dump/<user>/my-app/
# Open that URL → Copy All → paste into the chat

# Or one JSON blob of all files
GET /api/bundle/my-app
GET /dump/<user>/my-app/?format=json

# List / single file
GET /api/tree/my-app
GET /api/file/my-app?path=app.py

# Push whole folder (multipart) — like git push
POST /api/upload/my-app
# form: files[], message, mode=push|merge|replace

# Tree with hashes (for push preview)
GET /api/tree/my-app?meta=1

# Write selected files (requires token)
POST /api/commit/my-app
Authorization: Bearer <configured-token>
{ "message": "Update", "files": { "src/index.js": "console.log('hi')" }, "deletes": [] }
